1. POST /apis/v1/login/ | GET NOT ALLOWED

URI

_domain_/apis/v1/login/

Description

Validate user credentials to log in, if correct then returns key and username, else error response and code.

BODY (form-data)

credential

User’s Username or E-mail

password

User’s password associated with entered credentials

RESPONSES-

ERRORS

GET REQUEST TYPE

GET request are not allowed for this URI

HTTP_405_METHOD_NOT_ALLOWED

Response-

{"detail": "Method \"GET\" not allowed."}

INVALID BODY PARAMETERS

if body parameters mentioned above are not sent with the request

HTTP_400_BAD_REQUEST

Response-

{'error': "Invalid body parameter, the body must contain 'credential' and 'password'}

CREDENTIAL NOT FOUND

if provided credential does not exist in the database

HTTP_200_OK

Response-

{'error': "The Email/Username you entered doesn't belong to an account. Please check your Email/Username and try again."}

INVALID PASSWORD

if the provided password does not match with the credentials

HTTP_200_OK

Response-

{'error': "Invalid Passowrd"}

SUCCESS

LOGIN SUCCESSFUL

if credentials match correctly

HTTP_202_ACCEPTED

Response-

{

    "key": “Sudo-permanent token associated with the user, Used for all further requests”,

    "username": “User’s username”

}

Response Example-

{

    "key": "OuAjSaDtjCn6oEyhv6sNjsBjsdgyYdfjh848ShFSPXMzrFtiHx2E",

    "username": "indiagram"

}

2. POST /apis/v1/signup/choose-username/ | GET NOT ALLOWED

URI

_domain_/apis/v1/signup/choose-username/

Description

To check if the provided username is available for use, if not then return alternative similar username suggestions to choose from.

BODY (form-data)

username

User’s Desired Username

RESPONSES-

ERRORS

GET REQUEST TYPE

GET request are not allowed for this URI

HTTP_405_METHOD_NOT_ALLOWED

Response-

{"detail": "Method \"GET\" not allowed."}

INVALID BODY PARAMETERS

if body parameters mentioned above are not sent with the request

HTTP_400_BAD_REQUEST

Response-

{'error': "Invalid body parameter, the body must contain 'credential' and 'password'"}

USERNAME ALREADY USED

if the provided username is already assigned to another user

HTTP_200_OK

Response-

{

    "error": "The username {provided username} is not available",

    "suggestions": [“List of suggested alternative usernames”,]

}

 

Response Example-

{

    "error": "The username indiagram is not available",

    "suggestions": [

        "indiagramx",

        "indi_agram",

        "indi.agram",

        "ind_iagram",

        "ind.iagram",

        "in_diagram",

        "in.diagram",

        "indiagram427",

        "indiagram854",

        "indiagram376"

    ]

}

USERNAME NOT SAFE

if the provided username contains one of the restricted keywords, abusive/offensive word, or does not follow username nomenclature. 

HTTP_205_RESET_CONTENT

Response-

{'error': "Reason for rejection"}

Response Examples-

1. bitch   //abusive or an offensive word

{

    "error": "The username bitch is not available"

}

2. yo   // two-letter word

{

    "error": "The username yo is not available"

}

3. admin  //reserved keyword

{

    "error": "The username admin is not available"

}

4. indiagram.

{

    "error": "You can't end your username with a period."

}

5. .indiagram

{

    "error": "You can't start your username with a period."

}

6. india..gram

{

    "error": "You can't have more than one period in a row."

}

7. indiagramsocialmediaappmadeinindia

{

    "error": "Enter a name under 30 characters."

}

8. 69420

{

    "error": "Your username cannot contain only numbers."

}

9. indiagram$

{

    "error": "Username can only use letters numbers underscores and periods."

}

SUCCESS

USERNAME IS AVAILABLE

if the provided username is not assigned to any other user, is according to username nomenclature, not an offensive/abusive word, and not a reserved word.
